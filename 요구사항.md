# 요구사항.md (Requirements.md)

## 1. 프로젝트 개요
한국어는 단어의 마지막 글자에 받침이 있는지 없는지에 따라 조사가 변하는 특징이 있습니다. 본 프로젝트는 개발자가 복잡한 조건문을 작성할 필요 없이, 단 한 줄의 코드로 자연스러운 조사를 선택하고 결합해 주는 파이썬 오픈 소스 라이브러리 구축을 목표로 합니다.

## 2. 핵심 기능 요구사항

### 2.1 지능형 조사 자동 교정 (Smart Josa Selection)
* **단일 조사 입력 대응**: 사용자가 조사의 쌍(은/는)을 모두 적지 않고 `은`, `는`, `이`, `가` 등 하나만 입력하더라도 앞 단어의 받침 유무에 맞춰 자동으로 교정합니다.
    * 내부적으로 조사 쌍 매핑 테이블을 관리하여, 입력된 조사가 어떤 쌍(예: 은/는, 이/가, 을/를, 과/와, 아/야 등)에 속하는지 스스로 판단합니다.
    * 예: `attach("사과", "은")` -> `"사과는"` (자동 교정)
    * 예: `attach("책", "는")` -> `"책은"` (자동 교정)
* **다양한 표기 수용**: `은/는`, `(은)는`, `은는` 등 어떤 형식의 조사 쌍을 입력해도 동일한 규칙을 적용합니다.

### 2.2 판별 엔진 (Hangeul & Number Support)
* **한글 유니코드 분석**: 한글 유니코드 공식(`(ord(char) - 0xAC00) % 28`)을 활용해 종성(받침)의 인덱스를 계산합니다.
* **숫자 받침 판별**: 숫자로 끝나는 경우, 해당 숫자를 한글로 읽었을 때의 받침 유무를 판별합니다.
    * 받침 있음: 0(영), 1(일), 3(삼), 6(육), 7(칠), 8(팔)
    * 받침 없음: 2(이), 4(사), 5(오), 9(구)
* **'ㄹ' 받침 예외 처리**: '으로/로' 조사의 경우, 'ㄹ' 받침으로 끝나는 단어(예: 하늘, 칼, 7, 8)는 받침이 없는 경우와 동일하게 '로'를 선택하는 특수 로직을 포함합니다.

### 2.3 지원 조사 범위
* 기본 조사: `은/는`, `이/가`, `을/를`, `과/와`, `아/야`, `으로/로`
* 확장 조사: `이랑/랑`, `이나/나`, `이야/야`, `이나마/나마`, `이라야/라야` 등 '이'가 탈락/추가되는 모든 형태 포함.

## 3. 라이브러리 사용성 및 배포 요구사항

### 3.1 명확한 API 설계 (Explicit Intent)
* **Zero-Configuration**: 설치 후 별도의 설정이나 사전 로드 없이 `import` 즉시 사용할 수 있어야 합니다.
* **직관적 API**: `attach(단어, 조사)`를 기본으로 합니다.
* **템플릿 포맷팅**: 문장 전체를 처리하는 `format()` 기능을 제공하며, 개발자의 의도를 명확히 하기 위해 파이프(`|`) 기호를 사용합니다.
    * 예: `josa.format("{name|은} {item|를} 샀다", name="수박", item="빵")` 
    * 결과: `"수박은 빵을 샀다"`

### 3.2 경량화 및 호환성
* **의존성 최소화**: 외부 라이브러리 없이 파이썬 표준 라이브러리만 사용하여 가볍고 빠르게 동작합니다.
* **타입 힌팅**: 모든 함수에 Type Hinting을 적용하여 IDE에서 자동 완성이 원활하게 동작하도록 합니다.
* **PyPI 표준**: `pip install` 명령어로 즉시 설치 가능한 구조를 갖춥니다.

## 4. 예외 케이스 처리
* **비한글/비숫자 문자**: 영어, 특수문자 등으로 단어가 끝날 경우, 에러를 내지 않고 합리적인 기본값(보통 받침이 없는 경우의 조사 또는 원본 유지)을 반환하는 방어적 코드를 작성합니다.
* **빈 문자열**: 단어가 빈 문자열인 경우 입력된 조사를 그대로 반환하거나 적절한 예외 처리를 수행합니다.
